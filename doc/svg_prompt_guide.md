# SVG电路图绘制提示词指南

## 目标
为全屋智能讲解创建交互式SVG电路图，使用React组件实现，支持动态参数调整和动画效果。

## 技术要求
- 使用React编写SVG模块组件
- 所有组件需支持动态参数配置
- 组件间可通过连线建立连接关系
- 支持脉冲信号动画效果展示数据流向

## 组件规范

### 1. 电源组件 (Power Component)
**功能描述：** 模拟家庭供电系统，主要用于演示，能够连接多个零线和火线

**属性配置：**
- `name`: 电源名称（字符串）
- `status`: 供电状态（布尔值，true表示正常供电，false表示断电）

**视觉表现：**
- 显示电源名称
- 通过颜色或图标显示供电状态（绿色表示供电正常，灰色表示断电）
- 设计上要能清晰地连接多条线路（零线和火线）
- 外观简洁，重点突出供电功能而非复杂参数

### 2. 灯组件 (Light Component)
**功能描述：** 模拟各种类型的灯具，支持状态显示和参数调节

**属性配置：**
- `name`: 灯具名称（字符串）
- `status`: 开关状态（布尔值，true表示开启，false表示关闭）
- `onColor`: 开启时的颜色（十六进制颜色值，默认#FFD700）
- `offColor`: 关闭时的颜色（十六进制颜色值，默认#CCCCCC）
- `brightness`: 亮度级别（数值，范围0-100）
- `colorTemperature`: 色温（数值，范围2000-6500K）

**视觉表现：**
- 显示灯具名称
- 根据开关状态显示不同颜色
- 可通过亮度参数调整发光效果
- 可通过色温参数调整光色（暖光到冷光）

### 3. 开关组件 (Switch Component)
**功能描述：** 模拟各类智能开关，显示通断状态和其他工作状态

**属性配置：**
- `name`: 开关名称（字符串）
- `isOn`: 通断状态（布尔值，true表示导通，false表示断开）
- `additionalStatus`: 其他状态信息（字符串数组）
- `type`: 开关类型（字符串，如"零火版"、"单火版"、"显示面板"等）

**视觉表现：**
- 显示开关名称
- 通过明显视觉差异显示通断状态
- 以文字形式展示额外状态信息
- 不同类型开关具有不同的外观标识

### 4. 连线组件 (Wire Component)
**功能描述：** 连接各个智能设备，可视化不同类型线路

**属性配置：**
- `startPoint`: 起始点坐标（对象{x, y}）
- `endPoint`: 终止点坐标（对象{x, y}）
- `lineType`: 线路类型（枚举值："network"-网络信号线, "live"-火线, "neutral"-零线）
- `signalActive`: 是否激活脉冲信号（布尔值）
- `signalColor`: 信号颜色（十六进制颜色值，默认#00BFFF）
- `signalSpeed`: 信号速度（数值，范围1-10）
- `lineWidth`: 线条粗细（数值，默认2px）

**视觉表现：**
- 不同类型线路使用不同颜色和样式：
  - 网络信号线：蓝色线
  - 火线：红色线
  - 零线：蓝色线（较浅）
- 连接两个设备组件
- 支持脉冲信号动画（从一端流向另一端）
- 信号流动方向应清晰可见

## 交互要求

### 动态控制
- 所有组件参数应支持实时调整
- 状态变化应立即反映在视觉表现上
- 连线中的信号流响应设备状态变化

### 动画效果
- 脉冲信号沿连线流动
- 灯具开关时有淡入淡出效果
- 开关状态切换时有视觉反馈

## 示例场景

### 场景1：基础照明控制
```
[智能开关] --(脉冲信号)--> [LED灯]
```
- 开关开启时，连线中出现蓝色脉冲信号流向灯具
- 灯具接收信号后点亮，显示设定的亮度和色温

### 场景2：多路控制
```
[中央控制器]
    |
    |--(信号)--> [客厅开关] --(信号)--> [客厅灯]
    |
    |--(信号)--> [卧室开关] --(信号)--> [卧室灯]
```
- 中央控制器向各区域开关发送控制信号
- 各开关独立控制对应区域灯具
- 信号流向通过脉冲动画清晰展示

## 输出格式
- 提供完整的React组件代码
- 组件应具有良好的可复用性和可扩展性
- 代码注释完整，参数说明清晰
- 支持TypeScript类型定义（如果可能）

## 注意事项
- 保持视觉风格统一
- 确保动画流畅不卡顿
- 颜色搭配应符合科技感和专业性
